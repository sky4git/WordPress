== Description ==
Enhance your website performance by serving website media and content in a smart way.

You can serve your WordPress website media and contents directly from your cloud storage, such as Amazon S3 or Google Cloud Storage, easily using Media Storage to Cloud.

Simply sync your WordPress site with your cloud storage using Media Storage to Cloud and all of your media and other important files will be stored on your cloud storage.

Then serve your website media and content via your secured cloud portal and watch your website become faster and more efficient.

>It's an easy 3 step process:
>1. Connect your cloud storage with Media Storage to Cloud.
>2. Serve your website media and content directly from your cloud storage.
>3. Watch your website performance improve immediately

>  [documentation](https://rextheme.com/docs/wp-media-storage-to-cloud/).

== Why Use Media Storage to Cloud? ==
Media Storage to Cloud has a simple interface that allows you to sync your WordPress website files to your cloud storage in a single click.

>Fully compatible with Amazon S3 and Google Cloud Storage

**Easy to sync media and other files to cloud storage**
Authorize your cloud storage and assign your desired bucket. Then simply click on sync, and all of your media and other important files will be copied to your cloud storage in organized folders.

>Currently you can only send image files to cloud storage

**Serve your website media and content directly from cloud storage**
Simply turn on the option to serve media files from a bucket, and all of your website media and other file URLs will be changed to cloud storage URLs. No manual hassle and no time wasted.


**Auto-sync with cloud storage for changes in your media and files** 
You no longer have to log in to your cloud storage manually for every change you make regarding website media or files.

You can turn on the option so that any media or file you upload to your WordPress storage, will be automatically uploaded to your cloud storage.

Also, you have the option so that any media or file you delete from site, will be automatically deleted from your cloud storage as well.

Appsero SDK **does not gather any data by default.** The SDK only starts gathering basic telemetry data **when a user allows it via the admin notice**. We collect the data to ensure great user experience for all our users.

== Frequently Asked Questions ==
= 1. What cloud storages do your support? =
The plugin currently supports Amazon S3 and Google Cloud Storage.

= 2. Installation =
You can download the plugin from wordpress.org. Once you have downloaded the file, you can then go to your dashboard, under plugin, select Add New and upload the file. Then Install and activate the plugin. Once activated, on the left side under your dashboard, you will find the menu 'WP Cloud' at the bottom.

= 3. How does the plugin work? =
The plugin uses API from Amazon S3 or Google Cloud Storage to send media/files and set up sync.

= 4. What type of files are synced? =
Right now you can only upload image files, such as JPG, JPEG, PNG, SVG, etc, from your media library. However, we will soon allow the plugin to send other files such as PDF, ZIP, etc, in an upcoming update.

= 5. Can I connect CloudFront or any CDN? =
Yes, You can connect CloudFront or any CDN.

= 6. How many files can I upload to my cloud storage? =
Currently, the plugin allows 500 media files to be sent and synced with your cloud storage. (All size variations for a single image, together, is considered as 1 media file.)
